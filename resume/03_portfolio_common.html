네이버 해피빈 개편

해피빈 포트폴리오 개요
주소 https://happybean.naver.com/
레이아웃 코딩 방식 HTML, CSS
제작기간 4 weeks
제작참여도 마크업 100%
이슈 및 문제 해결

- 개발 확정 이후 마크업 논의가 진행되어 기존 구조를 최대한 유지하여 최소한의 구조 변경으로 UI 개편을 진행한 사례
- 일부 서비스는 유지해야하는 문제로 기존 css와의 충돌 및 사이드이팩트 문제의 어려움
- 구조 변경이 제한적인 부분은 디자인과 협업하여 해결
- phantomjs를 활용한 웹페이지 diff tool을 적용하여 사이드이펙트 문제 해결

파파고 웹

파파고웹 포트폴리오 개요
주소 https://papago.naver.com/
레이아웃 코딩 방식 HTML, CSS, react Framework
제작기간 4 weeks
제작참여도 마크업 100%
이슈 및 문제 해결

- react Framework를 사용하였으나, 컴퍼넌트 분리는 개발에서 진행한 아쉬움
- PC only 스펙으로 진행 중 반응형 스펙으로 변경되어 rework 발생이 많았음.
- 구글 웹 폰트를 사용, 중국어/일본어에서 지원하지 않는 글자 이슈 발생
- 지원하지 않는 글자에서만 언어별 자국 시스템 폰트 대응돌 수 있도록 font-family 순서 정렬 함.

- bootstrap 기반의 gird 레이아웃 사용
